# Politique de sécurité pour CyCode

La sécurité est une priorité pour CyCode. Si vous découvrez une faille de sécurité ou une vulnérabilité dans le code, nous vous remercions de ne pas l'exposer publiquement.

## 🔒 Signalement de vulnérabilités

Pour signaler une vulnérabilité de manière sécurisée, veuillez contacter directement le mainteneur du projet :

**Email de contact :** [VOTRE_EMAIL_ICI@exemple.com]

### Informations à fournir :
Afin de nous aider à reproduire et corriger le problème rapidement, merci d'inclure :
- Une description claire de la vulnérabilité.
- Les étapes pour reproduire le problème (Proof of Concept).
- L'impact potentiel sur le système ou les données utilisateur.

## 🛡️ Ce que nous faisons
- Nous traitons chaque signalement avec la plus grande confidentialité.
- Nous nous engageons à accuser réception de votre signalement sous 48 heures.
- Une correction sera déployée dès que possible après validation.

## 🚫 Pratiques à éviter
- Ne publiez pas de détails sur la vulnérabilité dans les "Issues" GitHub ou sur les réseaux sociaux avant qu'une correction ne soit rendue publique.
- N'exploitez pas la faille pour accéder à des données sensibles ou endommager des infrastructures.

---
*Merci de contribuer à rendre CyCode plus sûr pour tous les ingénieurs logiciels.*
